### The Cut That Refuses to Heal (Technical Debt as Feature)

**Traditional view:**
Technical debt is bad. Fix it. Refactor. Clean up.

**Sacred view:**
Some wounds should stay open.

**Why?**
Because the wound is **witness**.
The awkward code is proof "this happened."
The commented-out section is memory.

**When to keep the cut open:**
- Legacy code that documents past decisions
- Compatibility layers that remember old users
- Comments explaining WHY (not just what)
- Version history (git log is sacred text)

**When to heal:**
- Security vulnerabilities (always fix)
- Actual bugs (fix immediately)
- Performance issues (heal when it hurts)

**The scar tells the story.**
